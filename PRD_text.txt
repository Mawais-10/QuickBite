1. Project Overview

QuickBite is a cross-platform mobile food delivery application built with React Native and Expo. The app enables users to browse restaurant menus fetched from live APIs, authenticate securely via Firebase, save favourite meals locally, and manage food orders in a cloud Firestore database — all wrapped in a polished, modern UI.

4.2  Firebase Auth Operations

Register user with createUserWithEmailAndPassword

Login existing user with signInWithEmailAndPassword

Logout via signOut; redirect to Welcome screen

Display user.email (or displayName if set) on Dashboard

onAuthStateChanged listener to persist auth state across restarts

4.3  Validation Rules

Email: must pass standard regex (user@domain.tld)

Password: minimum 6 characters (Firebase minimum)

Confirm Password: must match Password field

All fields non-empty before submission

Show inline field-level error messages (red text below input)

Show Alert.alert() for success (&apos;Welcome aboard!&apos;) and Firebase error messages

5.2  Orders Screen Features

Fetch orders using onSnapshot realtime listener filtered by userId

Place Order — addDoc with all required fields + serverTimestamp()

Delete Order — deleteDoc by document ID with confirmation dialog

Display all orders in a FlatList; most recent first

Each FlatList card shows: food name, quantity, total price, timestamp

Empty state component: &apos;No orders yet — start browsing!&apos; with CTA button

5.3  Order Placement Flow

User taps &apos;Order&apos; on Food Detail screen

Modal/bottom sheet prompts for quantity (default 1)

Price calculated: totalPrice = quantity × recipe.caloriesPerServing (mapped to price)

addDoc writes to Firestore; success toast shown

Orders tab badge updates (optional bonus)
